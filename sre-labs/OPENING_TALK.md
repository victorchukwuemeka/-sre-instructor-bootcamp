# 10-Minute Opening Talk (Script)

## 0:00–1:00 — Welcome + Outcome
Good morning, everyone. This course is about one thing: reliability at scale. By the end, you’ll be able to design services that stay available even when failures happen, because failures are normal.

## 1:00–3:00 — What SRE Is
SRE is software engineering applied to operations. Instead of manual firefighting, we design systems and automation so reliability is predictable.

## 3:00–5:00 — The Core Promise (SLI/SLO/SLA)
We measure what matters (SLI), set a target (SLO), and sometimes put it in a contract (SLA). This is how we align engineering, product, and business.

## 5:00–7:00 — Error Budgets
Error budgets are how much failure we can accept. If we spend the budget, we pause risky changes and focus on reliability. It’s the balance between speed and stability.

## 7:00–9:00 — Observability + Automation
You can’t fix what you can’t see. Metrics, logs, and traces show you what’s happening. Automation reduces toil so engineers can focus on real improvements.

## 9:00–10:00 — Today’s Path
We’ll do hands-on labs for monitoring, incident response, and Kubernetes basics. You’ll leave with real skills you can apply immediately.
